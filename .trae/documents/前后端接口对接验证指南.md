# 前后端接口对接验证指南

## 接口对接概述

### 前端配置
- **开发服务器端口**: 5173
- **API 代理配置**: `/api` → `http://localhost:8080`
- **Axios baseURL**: `/api`

### 后端配置
- **服务器端口**: 8080
- **上下文路径**: `/api`
- **数据库**: MySQL 8.0

## 接口列表

### 1. 攻略接口

**接口路径**: `GET /api/guides`

**参数**:
- `tag` (可选): 标签筛选，如 "开荒攻略"、"全收集攻略"
- `keyword` (可选): 关键词模糊搜索

**请求示例**:
```bash
# 获取所有攻略
curl http://localhost:8080/api/guides

# 按标签筛选
curl "http://localhost:8080/api/guides?tag=开荒攻略"

# 关键词搜索
curl "http://localhost:8080/api/guides?keyword=名刀月隐"
```

**响应格式**:
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "title": "名刀月隐获取攻略",
      "summary": "本文详细介绍如何获取名刀月隐...",
      "content": "详细攻略内容...",
      "tags": "开荒攻略,武器攻略",
      "coverUrl": "https://...",
      "createdAt": "2024-01-01T00:00:00",
      "updatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

### 2. Boss 检查清单接口

**接口路径**: `GET /api/checklist`

**参数**: 无

**请求示例**:
```bash
curl http://localhost:8080/api/checklist
```

**响应格式**:
```json
{
  "code": 200,
  "data": {
    "bossStatusList": [
      {
        "boss": {
          "id": 1,
          "name": "接肢者",
          "difficultyLevel": 3
        },
        "isCompleted": false
      }
    ],
    "totalBosses": 10,
    "completedBosses": 3,
    "progressPercent": 30
  }
}
```

### 3. 切换 Boss 状态接口

**接口路径**: `POST /api/checklist/toggle`

**请求体**:
```json
{
  "bossId": 1
}
```

**请求示例**:
```bash
curl -X POST http://localhost:8080/api/checklist/toggle \
  -H "Content-Type: application/json" \
  -d '{"bossId": 1}'
```

**响应格式**:
```json
{
  "code": 200,
  "data": {
    "progressPercent": 40,
    "completedBosses": 4,
    "totalBosses": 10
  }
}
```

## 验证步骤

### 步骤 1: 启动后端服务
```bash
# 进入后端目录
cd backend

# 启动 Spring Boot 应用
mvn spring-boot:run

# 或运行 JAR 包
java -jar target/soul-game-db-0.0.1-SNAPSHOT.jar
```

### 步骤 2: 启动前端服务
```bash
# 进入前端目录
cd frontend

# 安装依赖（如果首次运行）
npm install

# 启动开发服务器
npm run dev
```

### 步骤 3: 测试接口

#### 测试攻略接口
1. 打开浏览器访问: `http://localhost:5173`
2. 打开浏览器开发者工具 → Network 面板
3. 刷新页面，查看是否有 `/api/guides` 请求
4. 检查响应状态码是否为 200
5. 检查响应数据格式是否正确

#### 测试标签筛选
1. 点击左侧"开荒攻略"按钮
2. 在 Network 面板查看请求参数是否包含 `?tag=开荒攻略`
3. 验证返回的攻略列表是否都是"开荒攻略"标签

#### 测试搜索功能
1. 在搜索框输入"名刀月隐"
2. 按回车键
3. 在 Network 面板查看请求参数是否包含 `?keyword=名刀月隐`
4. 验证返回的攻略列表是否包含关键词

#### 测试 Boss 检查清单
1. 查看右侧是否显示 Boss 列表
2. 查看进度条是否正确显示百分比
3. 点击 Boss 复选框
4. 验证进度条是否平滑涨动
5. 在 Network 面板查看是否有 `/api/checklist/toggle` 请求

## 常见问题排查

### 问题 1: 请求 404
**原因**: 后端服务未启动或端口配置错误

**解决方案**:
1. 确认后端服务正在运行
2. 检查后端端口是否为 8080
3. 检查 Vite 代理配置是否正确

### 问题 2: 请求超时
**原因**: 后端处理时间过长或网络问题

**解决方案**:
1. 检查数据库连接是否正常
2. 查看后端日志是否有错误
3. 增加超时时间配置

### 问题 3: CORS 跨域错误
**原因**: 后端未配置 CORS

**解决方案**:
检查后端 Controller 是否添加了 `@CrossOrigin(origins = "*")` 注解

### 问题 4: 数据格式错误
**原因**: 前后端数据结构不匹配

**解决方案**:
1. 检查后端返回的 JSON 格式
2. 检查前端接收数据的代码
3. 确保字段名称完全一致

- 请求处理日志
- 异常堆栈信息

## 性能优化建议

1. **使用缓存**: 对攻略列表添加前端缓存
2. **分页加载**: 如果攻略数量很大，添加分页功能
3. **图片优化**: 使用懒加载和图片压缩
4. **错误重试**: 添加自动重试机制
5. **乐观更新**: 对于用户操作立即更新 UI，后台异步请求
